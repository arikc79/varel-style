Варіант 1: З PostgreSQL (рекомендовано)
───────────────────────────────────────────
1. Встановити PostgreSQL з:
   https://www.postgresql.org/download/

2. Клонувати репозиторій:
   git clone https://github.com/arikc79/varel-style.git
   cd varel-style

3. Запустити SETUP_AND_RUN.bat
   (він автоматично налаштує все)

4. Готово! Сервер на http://127.0.0.1:8000

Варіант 2: Зі SQLite (для швидкого тесту)
───────────────────────────────────────────
1. Клонувати репозиторій:
   git clone https://github.com/arikc79/varel-style.git
   cd varel-style

2. Запустити SETUP_AND_RUN.bat
   (він виявить відсутність PostgreSQL)

3. Скрипт запропонує SQLite
   (автоматично додасть USE_SQLITE=True)

4. Готово! Сервер на http://127.0.0.1:8000

════════════════════════════════════════════════════════════
            📊 ПОРІВНЯННЯ PostgreSQL vs SQLite
════════════════════════════════════════════════════════════

PostgreSQL:
✅ Production ready
✅ Багатокористувацький
✅ Транзакції ACID
✅ Повна підтримка JSON
✅ Масштабування
✅ Резервне копіювання
❌ Потрібне встановлення

SQLite:
✅ Без встановлення
✅ Простота
✅ Швидкий старт
❌ Тільки один користувач
❌ Обмежені можливості
❌ НЕ для production

════════════════════════════════════════════════════════════
                🎯 РЕКОМЕНДАЦІЇ
════════════════════════════════════════════════════════════

Для розробки:
👉 Використовуйте PostgreSQL
   • Ідентичне production середовищу
   • Виявляйте проблеми раніше
   • Тестуйте всі можливості

Для швидкого тестування UI:
👉 Можна використати SQLite
   • Швидкий старт
   • Перевірка зовнішнього вигляду
   • Тестування frontend

Для production:
👉 ТІЛЬКИ PostgreSQL
   • SQLite НЕ підходить для production
   • Використовуйте Supabase, Render, або власний сервер
